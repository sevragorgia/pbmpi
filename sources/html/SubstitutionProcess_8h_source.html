27 // Substitution Process is the class gathering nearly all CPU-intensive methods of the program
28 //
29 // A substitution process object is aware of the "across site" dimension, but not of the "across branch" dimension (which is taken care of by PhyloProcess)
30 //
31 // most important accessors:
32 //
33 // int GetNsite() : number of aligned positions
34 // int GetNstate(int i) : number of states for the substitution process at site i
35 //
36 // int GetNrate(int i) : number of rate categories at site i
37 // double GetRate(int i) : gives the rate at site i (when sites are explicitly allocated to one of the rate categories)
38 //
39 // const double* GetProfile(int i) : equilibrium frequency profile at site i
40 
41 // how these rates and profiles are distributed across sites is not managed by SubstitutionProcess itself,
42 // but by its two super classes RateProcess and ProfileProcess
43 
44 // in addition, SubstitutionProcess implements a whole series of CPU intenstive methods for
45 // - propagating conditional likelihood vectors
46 // - sampling rates at each site proportionally to the conditional likelihoods
47 // - sampling states (amino-acids) at internal nodes of the tree
48 // - sampling substitution mappings along branches
49 
50 // since SubstitutionProcess is not aware of the overarching tree structure
51 // it does not deal with the overall structure of those computations
52 // which are in fact managed by the PhyloProcess subclass
53 
54 // In other words. PhyloProcess is responsible for the higher-level algorithmic part
55 // while SubstitutionProcess makes the "across-site" micro-managment part of the computation
56 
57 // Thus, for all these methods, PhyloProcess organizes the computations
58 // and repeteadly asks SubstitutionProcess to perform one pass over all sites for one of those computational items
59 // each time, providing SubstitutionProcess with the necessary information
60 // (e.g. length of the branch, state of the process at both ends for each site, etc.)
61 
62 // since phylogenomics is about using many aligned positions (large value of GetNsite())
63 // about everything that needs to be parallelized is here
64 // the only exception is "
